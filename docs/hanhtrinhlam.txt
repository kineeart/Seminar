02: Phát triển Frontend JavaScript
Bối cảnh

Contoso là một công ty bán các sản phẩm phục vụ cho nhiều hoạt động ngoài trời. Bộ phận marketing của Contoso muốn ra mắt một website mạng xã hội quy mô nhỏ (micro social media) để quảng bá sản phẩm tới khách hàng hiện tại và khách hàng tiềm năng.

Với vai trò là lập trình viên JavaScript, bạn sẽ xây dựng một ứng dụng frontend bằng React, giao tiếp với API backend Python.

Bạn muốn theo hướng nào?

FastAPI backend chưa chạy được vì thiếu uvicorn. Bạn cần cài đặt uvicorn trước khi khởi động API. Bạn có muốn tôi cài uvicorn cho môi trường Python trong thư mục complete/python không?


👉 CÓ — bạn nên cho phép cài uvicorn ✅
Vì FastAPI không thể chạy nếu thiếu uvicorn.

Vì sao cần uvicorn?

FastAPI = framework

uvicorn = ASGI server để chạy FastAPI

Thiếu uvicorn → API không thể start

Cách làm (ngắn gọn, đúng bước)
👉 Khi Copilot hỏi:

“Bạn có muốn tôi cài uvicorn… không?”
➡️ Chọn YES

Copilot sẽ tự cài:

pip install uvicorn


(hoặc pip install "uvicorn[standard]")

Nếu bạn muốn tự cài (chắc chắn 100%)

Trong terminal, vào đúng thư mục:

cd complete/python


Cài uvicorn:

pip install uvicorn


Kiểm tra:

uvicorn --version

Sau đó chạy FastAPI

Ví dụ phổ biến:

uvicorn main:app --host 0.0.0.0 --port 8000 --reload

👉 Cứ YES cho Copilot cài uvicorn, xong báo mình nếu có error (copy log là được), mình xử tiếp ngay.

FastAPI backend chưa chạy được vì Uvicorn không thể import module main trong complete/python. Lỗi này thường xảy ra khi file main.py không có hàm app hoặc tên module không đúng.

Bạn cần kiểm tra lại file main.py trong complete/python để đảm bảo có biến app (ví dụ: app = FastAPI()) và đúng tên module. Nếu cần tôi có thể kiểm tra và sửa giúp bạn. Bạn muốn tiếp tục không?

Chuẩn rồi 👍
Lỗi này KHÔNG phải lỗi FastAPI, mà là cơ chế bảo vệ của Python (PEP 668) trên Linux / Codespaces.

Mình giải thích ngắn gọn rồi làm đúng cách từ đầu, bạn chỉ cần copy–paste theo là chạy được.

1️⃣ Vì sao lỗi này xảy ra?

Thông báo chính:

error: externally-managed-environment


👉 Nghĩa là:

Python trong Codespaces / Linux bị quản lý bởi hệ điều hành

Không cho pip install trực tiếp vào system Python

Bắt buộc phải:

dùng virtual environment (venv)
❌ KHÔNG dùng pip install global

👉 Đây là đúng chuẩn, không phải máy bạn lỗi.
